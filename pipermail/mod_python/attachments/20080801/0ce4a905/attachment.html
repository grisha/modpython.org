<p class=MsoNormal>OperationalError: (2003, &quot;Can't connect to MySQL server

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>#perror 4<o:p></o:p></p>

<p class=MsoNormal>OS error code&nbsp;&nbsp; 4:&nbsp; Interrupted system call<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>I am using apache 2.2.6 with worker MPM , python 2.4.4 and
mod_python 3.3.1.<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>10.18.0.2 is a remote DB test server.<o:p></o:p></p>

<p class=MsoNormal>I upgraded MySQLdb adapter from 1.2.0 to 1.2.2 and got the
same error with both versions. The adapters are compiled with <o:p></o:p></p>

<p class=MsoNormal>mysqlclient_r , which is supposed to be thread-safe.<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>Then, I pointed MySQL connections to &nbsp;localhost, which

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>To get errors on localhost, I had to increase the request

<p class=MsoNormal>OperationalError: (2003, &quot;Can't connect to MySQL server

<p class=MsoNormal>127.0.0.1 is my development machine, and I have never seen
these errors before, until I ran this stress test today, bringing the machine
to load average 8.0. <o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>127.0.0.1 and 10.18.0.2 are two different machines, with
different kernels and even different MySQL versions, so it is the &nbsp;client machine
that is a problem.<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

<p class=MsoNormal>I have written a single-threaded stand-alone python loop,
