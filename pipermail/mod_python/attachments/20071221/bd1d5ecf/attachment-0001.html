Indeed it does need to get the directory listing.  Multiview tries to
match inexisting requests with the best match it can find in a
directory.  Therefore, it needs to get the listing and parse it. 
Example:  <a class="moz-txt-link-freetext" href="http://localhost/mypage">http://localhost/mypage</a>  may be redirected to
<a class="moz-txt-link-freetext" href="http://localhost/mypage.php">http://localhost/mypage.php</a> if mypage don't exist.  However, if 
mypage.xhtml, mypage.html, mypage.html3, mypage.psp, mypage.php all
exists, it's hard to know which file will be picked.<br>
