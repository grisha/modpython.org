mode is a bad idea as only option is to have a copy of each in every<br>
process. So, if using one Apache for many Django instances, better off<br>
using daemon mode as gives better control over each application and<br>
can provision less processes for Django instances which are less used<br>
so overall memory is less. You can also make use of multithreading<br>
where appropriate, that is, can handle greater capacity requests where<br>
most of the time requests are stuck waiting on databases. Your code<br>
though then has to be thread safe.<br>
