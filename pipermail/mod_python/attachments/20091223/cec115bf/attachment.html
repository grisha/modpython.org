As documented in:<br>
<br>
  <a href="http://blog.dscpl.com.au/2009/03/load-spikes-and-excessive-memory-usage.html" target="_blank">http://blog.dscpl.com.au/2009/03/load-spikes-and-excessive-memory-usage.html</a><br>
<br>
the OP is having problems because of how embedded mode Python<br>
applications work in Apache. This may be  exacerbated by using prefork<br>
MPM with Apache. Same issues come up with mod_wsgi embedded mode as<br>
with mod_python if you do not configure your Apache correctly for fat<br>
Python web applications. If you are limited in being able to do that<br>
because Apache is heavily used for static file serving or mod_php, you<br>
really should use mod_wsgi daemon mode instead as described in that<br>
post.<br>
<br>
Why daemon mode will help is fixed number of persistent processes and<br>
more specifically defaulting to one process, so you only incur startup<br>
cost once and not for many processes, or in case of embedded mode also<br>
not when the processes get recycled as is default for Apache.<br>
