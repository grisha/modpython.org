Ignore that example and try something simpler, if you&#39;re merely testing<br>
Publisher functionality.<br>
<br>
Create a file named hello.py:<br>
<br>
def bruce(req):<br>
 &nbsp; &nbsp;return &quot;Hello, Bruce!&quot;<br>
<br>
Now point your browser at the *function* in the module:<br>
<br>
<a href="http://host/hello.py/bruce" target="_blank">http://host/hello.py/bruce</a><br>
<br>
Once you get that working, add more sophisticated code. Publisher will<br>
inspect the string you return and set the Content-Type to &quot;text/plain&quot;<br>
if it&#39;s plaintext and &quot;text/html&quot; if it looks like an HTML page:<br>
<br>
def bruce(req):<br>
 &nbsp; &nbsp;return &quot;&lt;html&gt;&lt;body&gt;&lt;h1&gt;Hello, Bruce!&lt;/h1&gt;&lt;/body&gt;&lt;/html&gt;&quot;<br>
<br>
Make sure this is working before you try anything more sophisticated.<br>
Then try importing some basic modules and returning output you create<br>
with them. Troubleshooting smtplib at this point is a waste of time, as<br>
there are any number of reasons it can fail, including the presence of<br>
AppArmor or SELinux.<br>
