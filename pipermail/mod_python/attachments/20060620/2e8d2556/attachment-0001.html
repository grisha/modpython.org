<p class=MsoNormal><font size=3 face="Times New Roman"><span style='font-size:
12.0pt'>There have been questions from Karl Kobata and John Cartwright and I
may have a partial answer.<br>
<br>
In particular, one of the problems that I spent a little time over was the use
of the Apache &lt;Directory&gt; tag to control mod_python configuration.<br>
<br>
The example in the mod_python tutorial is slightly ambiguous, particularly as
it is written from a Unix / Linux perspective. I suspect that this is
bog-standard Apache configuration rather than anything peculiar to mod_python
but I guess that people are coming fairly new to both at the same time (as was
I).<br>
<br>
The directory in the &lt;Directory&gt; tag has to be an absolute Windows
directory that should include a drive letter. This will have to map to a
directory under the Apache root directory. I was expecting the reference to be
relative to the Apache root directory.<br>
<br>
For example, suppose that you have an Apache configuration with the Apache root
set to E:\ApacheRoot and you have a sub-directory testdir in which you have
placed mod_python sources. Then you can refer to URIs in this directory from
the browser as <a href="http://yourdomain/testdir/test.py">http://yourdomain/testdir/test.py</a>
if you have a set of Apache directives such as the folliowing<br>
</span></font><font size=2 face=Arial><span style='font-size:10.0pt;font-family:
Arial'>&lt;Directory &quot;E:\ApacheRoot\testdir&quot;&gt;</span></font> <o:p></o:p></p>
