<FONT COLOR="#000000">Michael Guerrero wrote:</FONT>

<FONT COLOR="#000000">&gt; Thanks for the config fix Graham.  Also, you said &quot;if you must use </FONT>
<FONT COLOR="#000000">&gt; SetHandler...&quot;.  I'm only using it because the examples showed it.  Is </FONT>
<FONT COLOR="#000000">&gt; it preferable to use AddHandler instead?  Pros/Cons?</FONT>

<FONT COLOR="#000000">AddHandler results in less conflicts, but limits you to including the </FONT>
<FONT COLOR="#000000">extension of filename.py in the URL. I typically enable it for the </FONT>
<FONT COLOR="#000000">entire DocumentRoot, just so I can place a published module anywhere </FONT>
<FONT COLOR="#000000">(but rarely use this approach).</FONT>

<FONT COLOR="#000000">SetHandler requires you to commit a directory to serve a special </FONT>
<FONT COLOR="#000000">purpose, but offers extra mod_python features and a degree of future </FONT>
<FONT COLOR="#000000">proofing. I develop the majority of my applications this way.</FONT>

<FONT COLOR="#000000">For example, with SetHandler and mod_python.publisher, you can have an </FONT>
<FONT COLOR="#000000">URL like this:</FONT>

<FONT COLOR="#000000">  <A HREF="http://host/dir/foo.py/index">http://host/dir/foo.py/index</A></FONT>

<FONT COLOR="#000000">Where index() is a function in /dir/foo.py. It can also be accessed like </FONT>
<FONT COLOR="#000000">this:</FONT>

<FONT COLOR="#000000">  <A HREF="http://host/dir/foo/">http://host/dir/foo/</A></FONT>

<FONT COLOR="#000000">When your boss realizes that he can replace you with two young PHP </FONT>
<FONT COLOR="#000000">programmers for the same price, they can replace your code with:</FONT>

<FONT COLOR="#000000">  <A HREF="http://host/dir/foo/index.php">http://host/dir/foo/index.php</A></FONT>

<FONT COLOR="#000000">With the proper DirectoryIndex, this is also accessible via:</FONT>

<FONT COLOR="#000000">  <A HREF="http://host/dir/foo/">http://host/dir/foo/</A></FONT>

<FONT COLOR="#000000">So a lot of links won't need to be updated, no need for mod_rewrite, and </FONT>
<FONT COLOR="#000000">you can take a better job at Google with good references, to boot.</FONT>

<FONT COLOR="#000000">I put my stylesheets in a /css directory in the DocumentRoot, and all of </FONT>
<FONT COLOR="#000000">my pages share them, regardless of the language (HTML, Python, PHP, </FONT>
<FONT COLOR="#000000">Perl, etc.). Consider using this approach while you learn Publisher. </FONT>
<FONT COLOR="#000000">Unlike PHP, where you can deploy an entire application as a </FONT>
<FONT COLOR="#000000">self-contained directory, Publisher requires a different approach when </FONT>
<FONT COLOR="#000000">using SetHandler. You can hack the conf file to some extent, but it's </FONT>
<FONT COLOR="#000000">worth checking out alternate (and sometimes simpler) deployment </FONT>
<FONT COLOR="#000000">strategies that might be more portable.</FONT>
