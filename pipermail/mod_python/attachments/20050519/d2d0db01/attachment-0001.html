Then hereby I'll make it a feature request:<BR>
<BR>
I'd like to have a method to catch the html code as produced by the run() method of the psp.PSP object for use in e.g. an e-mail body, instead of having the run() method automatically pass it to the browser.<BR>
<BR>
I mean: normally one does something like:<BR>
page = psp.PSP(req,&nbsp; &quot;template.psp&quot;)<BR>
page.run({&quot;spam&quot;: 1, &quot;eggs&quot;: 5})<BR>
and the page is converted to html, and displayed in the browser. I'd like to use this templating for creating an e-mail, e.g.:<BR>
<BR>
page = psp.PSP(req,&nbsp; &quot;template.psp&quot;)<BR>
body = page.create({&quot;spam&quot;: 1, &quot;eggs&quot;: 5})<BR>
<BR>
Which basically would do exactly the same as the run() module, except for having the html code as output.<BR>
