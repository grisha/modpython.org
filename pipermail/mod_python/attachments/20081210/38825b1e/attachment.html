<br>
</div></div>If you use disable(), your filter will never be called for that<br>
request (input or output as appropriate) again.<br>
<br>
If you use pass_on(), then it is passing on data available to that<br>
specific call to your filter function.<br>
<br>
The distinction exists because your filter function can be called<br>
multiple times for a request. The latter is allowing you to pass on<br>
the data for that call, but still allowing you the option to process<br>
data in a subsequent call of the filter function.<br>
<br>
The disable() call is used internally to mod_python to ensure that<br>
filter function is never called again for a specific request if the<br>
filter function itself raises an exception of some sort. If it doesn&#39;t<br>
do this you could end up with a loop if the error in filter function<br>
actually generates output, for which it would then call the filter<br>
function again, etc etc.<br>
<br>
FWIW, if you are concerned about performance in any way, then you<br>
shouldn&#39;t be using mod_python to implement Apache input/output<br>
filters. Yes you can write filters with mod_python, but it will impact<br>
performance. For best performance you should write filters in C code<br>
as proper Apache module.<br>
