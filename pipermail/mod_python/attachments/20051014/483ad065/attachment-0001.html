PythonImport makes sure that the module is imported when the
interpreter is started. So you'll find it in sys.module. However, to
see it amongst your module's global variables, you need it import it in
your module. This is exactly the same in a standard Python interpreter
: it's not because a module has been imported by a module that it is
present in all the other modules' global variables.<br>
<br>
So your published module should be :<br>
<br>
import my_test<br>
<br>
def t(req):<br>
&nbsp;&nbsp;&nbsp; return globals()<br>
<br>
Note two things :<br>
<br>
1) I've renamed test to my_test, because test is a standard Python module and trouble may ensue if you use this module name.<br>
2) str() around globals is not needed IIRC. The publisher will apply it. But that's a detail.<br>
