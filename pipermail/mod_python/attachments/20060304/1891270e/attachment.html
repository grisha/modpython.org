    <FONT SIZE="2"><FONT COLOR="#0000ff">I guess a good place to start is to&nbsp;note that mod python is not, in itself, a web content framework nor a web application framework - it is an interface between the Apache HTTP server and the Python programming language.&nbsp; The true beauty of mod python is, in my opinion, is that you can use it for, well, any kind of HTTP-based service you need.&nbsp; Anything written in Python, or anything that will &quot;speak nicely&quot; to Python, can be implemented over HTTP with mod python.</FONT></FONT>
</BLOCKQUOTE>
<BLOCKQUOTE TYPE=CITE>
    <FONT COLOR="#000000">&nbsp;</FONT>
</BLOCKQUOTE>
<BLOCKQUOTE TYPE=CITE>
    <FONT SIZE="2"><FONT COLOR="#0000ff">As an example, I use mod python for an extremely simple and straight-forward content management system.&nbsp; My content is XML data that is transformed via XSLT into web pages.&nbsp; When a request arrives at the server, a very simple mod python handler maps the request URL to a pair of&nbsp;XML and&nbsp;HTML files. if the file timestamp on the&nbsp;HTML file is newer than the&nbsp;XML file then the HTML&nbsp;file is served as-is.&nbsp; If the XML file is newer than the HTML file, an external XSLT processor is called (LIBXSLT, in my case) to refresh the HTML file and serve it.</FONT></FONT>
</BLOCKQUOTE>
<BLOCKQUOTE TYPE=CITE>
    <FONT COLOR="#000000">&nbsp;</FONT>
</BLOCKQUOTE>
<BLOCKQUOTE TYPE=CITE>
    <FONT SIZE="2"><FONT COLOR="#0000ff">I wrote this little content management system because I did not like any of the &quot;out-of-the box&quot; CMS systems out there because they seemed much more complicated than my needs.&nbsp; But if you truly want a&nbsp; full-bore content management and delivery system, like CherryPy&nbsp;for example,&nbsp;you can do that, too.</FONT></FONT>
