<DIV dir=ltr align=left><SPAN class=251153920-04032006><FONT face=Arial 
color=#0000ff size=2>I guess a good place to start is to&nbsp;note that mod 
python is not, in itself, a web content framework nor a web application 
framework - it is an interface between the Apache HTTP server and the Python 
programming language.&nbsp; The true beauty of mod python is, in my opinion, is 
that you can use it for, well, any kind of HTTP-based service you need.&nbsp; 
Anything written in Python, or anything that will "speak nicely" to Python, can 
be implemented over HTTP with mod python.</FONT></SPAN></DIV>
<DIV dir=ltr align=left><SPAN class=251153920-04032006><FONT face=Arial 
color=#0000ff size=2></FONT></SPAN>&nbsp;</DIV>
<DIV dir=ltr align=left><SPAN class=251153920-04032006><FONT face=Arial 
color=#0000ff size=2>As an example, I use mod python for an extremely simple and 
straight-forward content management system.&nbsp; My content is XML data that is 
transformed via XSLT into web pages.&nbsp; When a request arrives at the server, 
a very simple mod python handler maps the request URL to a pair of&nbsp;XML 
and&nbsp;HTML files. if the file timestamp on the&nbsp;HTML file is newer than 
the&nbsp;XML file then the HTML&nbsp;file is served as-is.&nbsp; If the XML file 
is newer than the HTML file, an external XSLT processor is called (LIBXSLT, in 
my case) to refresh the HTML file and serve it.</FONT></SPAN></DIV>
<DIV dir=ltr align=left><SPAN class=251153920-04032006><FONT face=Arial 
color=#0000ff size=2></FONT></SPAN>&nbsp;</DIV>
<DIV dir=ltr align=left><SPAN class=251153920-04032006><FONT face=Arial

not like any of the "out-of-the box" CMS systems out there because they seemed 
much more complicated than my needs.&nbsp; But if you truly want a&nbsp; 
full-bore content management and delivery system, like CherryPy&nbsp;for
