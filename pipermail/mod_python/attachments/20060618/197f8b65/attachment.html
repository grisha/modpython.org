<HTML><BODY style="word-wrap: break-word; -khtml-nbsp-mode: space; -khtml-line-break: after-white-space; "><BR><DIV><DIV>On 18/06/2006, at 7:06 AM, ian McDowall wrote:</DIV><BR class="Apple-interchange-newline"><BLOCKQUOTE type="cite">There have been questions from Karl Kobata and John Cartwright and I may have a partial answer.<BR> <BR> In particular, one of the problems that I spent a little time over was the use of the Apache &lt;Directory&gt; tag to control mod_python configuration.<BR> <BR> The example in the mod_python tutorial is slightly ambiguous, particularly as it is written from a Unix / Linux perspective.  I suspect that this is bog-standard Apache configuration rather than anything peculiar to mod_python but I guess that people are coming fairly new to both at the same time (as was I).<BR> <BR> The directory in the &lt;Directory&gt; tag has to be an absolute Windows directory that should include a drive letter.  This will have to map to a directory under the Apache root directory.  I was expecting the reference to be relative to the Apache root directory.<BR> <BR> For example, suppose that you have an Apache configuration with the Apache root set to E:\ApacheRoot and you have a sub-directory testdir in which you have placed mod_python sources.  Then you can refer to URIs in this directory from the browser as <A href="http://yourdomain/testdir/test.py">http://yourdomain/testdir/test.py</A> if you have a set of Apache directives such as the folliowing<BR> <FONT face="Arial" size="2"><SPAN style="font-size: 10pt; font-family: Arial;">&lt;Directory "E:\ApacheRoot\testdir"&gt;</SPAN></FONT><P style="margin-left: 0.25in; text-indent: 0.25in;"><FONT face="Arial" size="2"><SPAN style="font-size: 10pt; font-family: Arial;">      AddHandler mod_python .py</SPAN></FONT></P><P style="margin-left: 0.25in; text-indent: 0.25in;"><FONT face="Arial" size="2"><SPAN style="font-size: 10pt; font-family: Arial;">      PythonHandler mptest</SPAN></FONT></P><P style="margin-left: 0.25in; text-indent: 0.25in;"><FONT face="Arial" size="2"><SPAN style="font-size: 10pt; font-family: Arial;">      PythonDebug On </SPAN></FONT></P><P style="margin-left: 0.25in; text-indent: 0.25in;"><FONT face="Arial" size="2"><SPAN style="font-size: 10pt; font-family: Arial;"> &lt;/Directory&gt;</SPAN></FONT></P> In this case, my confusion was to assume that I would have a tag as follows:<BR> <FONT face="Arial" size="2"><SPAN style="font-size: 10pt; font-family: Arial;">&lt;Directory "/testdir"&gt;</SPAN></FONT> <BR> ...<BR> <BR> BTW, if you are really struggling with set-up, I got some initial progress by omitting the &lt;Directory&gt; tag and just having the AddHandler etc. at the root level.  Of course, this is not wise for a real implementaton but, if you are trying to find out if your mod_python installation is OK then it gave me some information.  of course, I had to put the .py file in the Python Lib directory in the installation so it could be imported but that did get me some basics working.<BR> <BR> I hope that this helps.  if it is misleading or irrelevant then please ignore it and accept my apologies.<BR></BLOCKQUOTE><BR></DIV><DIV>The best way to avoid the confusion is not to set it up in the main Apache configuration</DIV><DIV>file but do it in a specific directory within a .htaccess file. That way the whole issue of</DIV><DIV>how to set Directory directive goes away. Where mod_python is already installed and</DIV><DIV>you just want to start playing, it also avoids the whole issue of pestering some administrator</DIV><DIV>to fiddle with the main Apache configuration unnecessarily. This is why the instructions</DIV><DIV>I post a reference to occasionally use a .htaccess file. It is generally easier to get going</DIV><DIV>and shows more problems in setup more quickly and more easily.</DIV><DIV><BR class="khtml-block-placeholder"></DIV><DIV>Those alternate instructions again are:</DIV><DIV><BR class="khtml-block-placeholder"></DIV><DIV>  <A href=
