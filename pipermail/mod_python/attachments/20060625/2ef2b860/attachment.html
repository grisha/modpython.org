<br><font size=2 color=#000080 face="Arial">Initial thought. &nbsp;To compile
and install 3.2.8, I believe needs flex 2.5.33 or newer. &nbsp;RHN has
flex 2.5.4. &nbsp;you will need to make sure that this is current. &nbsp;I
would suggest when using the .configure, that options for flex, python,
and apxs be fully specified to insure that you know which versions have
be used to build mod_python.</font>
<br><font size=2 color=#000080 face="Arial">&nbsp;</font>
<br><font size=2 color=#000080 face="Arial">It may be easier to start with
what RHN provides for their servers. &nbsp;ES packages should work on AS.
&nbsp;The latest released for ES is python 2.3.4-14.x and mod_python 3.1.3-5.x.</font>
<br><font size=2 color=#000080 face="Arial">- rename or move flex, and
python binaries if applicable from /usr/local/bin . &nbsp;if the default
paths have not been modified, this directory will be searched before /usr/bin.</font>
<br><font size=2 color=#000080 face="Arial">- install python first – “up2date
python” – if up2date fails, one strong possibilities is that the python
it files does not have the correct scripts in the site-packages directory
under the pythonx.x directory. &nbsp;Next, using this command will also
resolve any dependencies needed to make python work. &nbsp;All should be
done as root.</font>
<br><font size=2 color=#000080 face="Arial">- next install mod_python –
“up2date python” – this should address installing and resolving dependencies.</font>
<br><font size=2 color=#000080 face="Arial">- check install and version</font>
<br><font size=2 color=#000080 face="Arial">- “python –V” – see version
above</font>
<br><font size=2 color=#000080 face="Arial">- “which python” – should
show “/usr/bin/python”</font>
<br><font size=2 color=#000080 face="Arial">- check if mod_python.so installed
in “/etc/httpd/modules”</font>
<br><font size=2 color=#000080 face="Arial">- add the required changes
into the .conf file - /etc/httpd/conf.d/python.conf.</font>
<br><font size=2 color=#000080 face="Arial">&nbsp; &nbsp; &nbsp; &nbsp;
&nbsp; &nbsp; - add the load directive for mod_python.so</font>
<br><font size=2 color=#000080 face="Arial">&nbsp; &nbsp; &nbsp; &nbsp;
&nbsp; &nbsp; - test if the server will read the .so file correctly</font>
<br><font size=2 color=#000080 face="Arial">&nbsp; &nbsp; &nbsp; &nbsp;
&nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; - stop the server
– “/sbin/service httpd stop”</font>
<br><font size=2 color=#000080 face="Arial">&nbsp; &nbsp; &nbsp; &nbsp;
&nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; - execute “/usr/sbin/httpd
–X –e DEBUG – the list of .so files loading should scroll by, mod_python.so
should be in this list.</font>
<br><font size=2 color=#000080 face="Arial">- assuming no errors/warnings
to this point, restart the web server – “/sbin/service httpd restart</font>
<br><font size=2 color=#000080 face="Arial">- now you will need to add
into “python.conf” the &lt;Directory…&gt; directives, the restart server.</font>
<br><font size=2 color=#000080 face="Arial">&nbsp;</font>
