You have asked and answered your own question.&nbsp; A framework usually provides components that make up a &quot;best practices&quot; architecture of a web application such as an MVC skeleton to work within, templating, and ORM.&nbsp; You are correct that most frameworks choose one particular implementation of each of those components and if it is not to your liking then you should choose another framework or roll-your-own (or join the development team of the framework and code a plugin or extension.)&nbsp; What frameworks do is encourage code re-use.&nbsp; Instead of rolling-your-own you get the plumbing for free.&nbsp; If you are going to choose to use straight-up mod_python I can guarantee that after working on a project or two you will spend time slowly developing your own framework so that for each request you will not have to cut and paste mundane code that generically handles the request and does often repeated tasks.
