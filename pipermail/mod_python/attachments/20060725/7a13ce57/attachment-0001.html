The problem is not related to the choice of technology. It is about accepting data from the web and using it in an executable context.<br><br>If the command you pass are built from data sent over the web, there is a chance that some malicious data can execute dangerous code. Therefore, you have to make sure that any data sent over the web (in forms or query parameters) is properly escaped when included in executable code (command line parameters, SQL requests, etc.).
<br><br>For more information, see the classical &quot;SQL Injection&quot; problem.<br><br><a href="http://en.wikipedia.org/wiki/SQL_injection">http://en.wikipedia.org/wiki/SQL_injection</a><br><br>Regards,<br>Nicolas<br><br>
