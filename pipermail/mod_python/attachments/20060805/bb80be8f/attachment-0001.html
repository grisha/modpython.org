Hi Faizan,<br><br>Here is an answer I gave a few month ago about a similar problem.<br><br>&quot;&quot;&quot;<br>One of the possible cause of your problem is that Apache does find<br>mod_python.so but not one of the <span id="st" name="st" class="st">
DLL</span> it relies on. Unfortunately the<br>error reporting in this case is quite scarce and misguiding.<br><br>The candidates missing DLLs are python24.<span id="st" name="st" class="st">dll</span> and the Microsoft Runtime
<br>DLLS (normally msvcr71.<span id="st" name="st" class="st">dll</span> and msvcp71.<span id="st" name="st" class="st">dll</span>).<br><br>Make sure that your Python directory and the runtime DLLs can be found<br>on your 
<span id="st" name="st" class="st">PATH</span>. The runtime DLLs are normaly copied in your Python<br>directory, so putting the Python directory on your <span id="st" name="st" class="st">PATH</span> variable should<br>do the trick.
<br><br>Also, make sure that the user which is running Apache has its <span id="st" name="st" class="st">PATH</span><br>variable properly set. The best way to do this is to define it at the<br>system-wide level, but logging it under the user and defining the
<br>variable at the user level is also possible.<br>&quot;&quot;&quot;<br><br>Hopefully we'll get the mod_python Wiki up and running real soon now, so we'll be able to add a bit of documentation on the subject.<br><br>Regards,
