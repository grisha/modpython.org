<br>
</div>You should though from memory only be accessing session object as<br>
&#39;session&#39; and not &#39;req.session&#39;. This is because PSP will look for the<br>
fact that &#39;session&#39; is used and use that as a basis of creating it for<br>
you.<br>
<br>
It is possible that the way you are doing this is deadlocking the<br>
request on itself as you can&#39;t have one request create two session<br>
objects.<br>
<br>
Once it is deadlocked like that, even if you close the browser it says<br>
deadlocked and so further requests will also be locked out.<br>
<br>
Thus, fix the code, restart Apache to get rid of deadlock and start over.<br>
